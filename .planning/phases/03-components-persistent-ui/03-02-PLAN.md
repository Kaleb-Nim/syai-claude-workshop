<!-- Slidev composables and global-layer contract -->

`global-bottom.vue` is a Slidev global layer file: Slidev auto-loads it from project root and renders it once, persisted across slide navigation. (Doc: sli.dev/custom/directory-structure â†’ "Global Layers".)

`useNav()` from Slidev â€” common shape (verified across Slidev v52.x):
```ts
const {
  currentSlideNo,   // Ref<number>  â€” 1-based current slide
  total,            // ComputedRef<number>
  slides,           // Ref<SlideInfo[]>  â€” each has .no (1-based) + .meta with .frontmatter + .meta.slide
  go,               // (no: number) => void
  next,
  prev,
} = useNav()
```
Each entry in `slides.value` exposes the slide's frontmatter at `slide.meta.slide.frontmatter` (and the title at `slide.meta.slide.title` when parseable). `layout` is one of the standard frontmatter keys.

Slidev exposes `useNav` from the auto-imported runtime â€” the canonical import in user code is:
```ts
import { useNav } from '@slidev/client/composables/useNav'
```
If that path does not resolve at build time, the alternate working import in current Slidev is:
```ts
import { useNav } from '#slidev/nav'
```
Try the first; fall back to the second only if the build errors out with "module not found".

Vue Router (provided by Slidev) â€” `useRoute()` from `vue-router` gives the current path so we can hide the sidebar in presenter/overview routes:
```ts
import { useRoute } from 'vue-router'
const route = useRoute()
const isPresenter = computed(() => route.path.startsWith('/presenter'))
```
</interfaces>

**Why filter sections instead of listing every slide:** A flat list of every slide produces a 30-50 entry sidebar that competes with deck content. Phase 4 will add 9 chapter sections (intro, subagents, hooks, MCP, slash, skills, plugins, agent SDK, outro), so a section-only list stays usable. Slides marked `layout: section` are the canonical section anchors per Slidev convention. The very first slide (cover) is included as "Start" so participants can always jump home.

**Why presenter-mode guard via `v-if`, not just CSS:** CSS-only hiding (`display: none`) leaves the component in the Vue tree where its imports/reactives still run. A `v-if` check on `route.path` removes it entirely â€” zero risk of layout regression in presenter chrome.

```vue
<script setup lang="ts">
import { computed } from 'vue'
import { useRoute } from 'vue-router'
// Primary import path. If the bundler reports module-not-found at build,
// switch to: import { useNav } from '#slidev/nav'
import { useNav } from '@slidev/client/composables/useNav'

const route = useRoute()
const isHiddenRoute = computed(() => {
  const p = route.path
  return p.startsWith('/presenter')
    || p.startsWith('/overview')
    || p.startsWith('/print')
})

const { currentSlideNo, total, slides, go } = useNav()

interface SectionEntry { no: number; label: string }

const sections = computed<SectionEntry[]>(() => {
  const out: SectionEntry[] = []
  const all = slides.value ?? []
  if (all.length > 0) {
    out.push({ no: 1, label: 'Start' })
  }
  for (const s of all) {
    const fm = (s as any)?.meta?.slide?.frontmatter ?? {}
    if (fm.layout === 'section' && (s as any).no !== 1) {
      const title = (s as any)?.meta?.slide?.title?.trim?.()
      out.push({
        no: (s as any).no,
        label: title && title.length > 0 ? title : `Section ${(s as any).no}`,
      })
    }
  }
  return out
})

const activeSectionNo = computed(() => {
  const cur = currentSlideNo.value
  let active = 1
  for (const sec of sections.value) {
    if (sec.no <= cur) active = sec.no
  }
  return active
})

function jump(no: number) {
  go(no)
}
</script>

<template>
  <aside
    v-if="!isHiddenRoute"
    class="deck-sidebar"
    aria-label="Deck navigation"
  >
    <div class="deck-sidebar-progress">
      Slide {{ currentSlideNo }} / {{ total }}
    </div>
    <ul class="deck-sidebar-sections">
      <li
        v-for="sec in sections"
        :key="sec.no"
      >
        <button
          type="button"
          class="deck-sidebar-link"
          :class="{ 'is-active': sec.no === activeSectionNo }"
          @click="jump(sec.no)"
        >
          {{ sec.label }}
        </button>
      </li>
    </ul>
  </aside>
</template>
```
