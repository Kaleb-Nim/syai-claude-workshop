Create `components/CopyButton.vue` as a self-contained button. It receives the source text via a `source` prop (string) and exposes a click handler that:
1. Calls `navigator.clipboard.writeText(props.source)`.
2. On success, sets local ref `copied.value = true`, then after 2000ms (window.setTimeout) sets it back to false.
3. On failure (Promise rejects, e.g. permission denied), logs `console.warn('[CopyButton]', err)` and leaves state at false.

The button label toggles between "Copy" (default) and "Copied" (when `copied === true`). Use `aria-label="Copy code"` always (not toggled). Use `<button type="button">` so it never submits a form.

Use `<script setup lang="ts">`. Do NOT import any external clipboard library. Do NOT use `document.execCommand` (deprecated).

Styling: minimal — class `copy-button` only. The actual visual rules (position, color, padding) live in `style.css` so they stay under the `.slidev-layout` scope contract (D-14). Inside the .vue, only set semantic structure:

```vue
<script setup lang="ts">
import { ref } from 'vue'

const props = defineProps<{ source: string }>()
const copied = ref(false)

async function copy() {
  try {
    await navigator.clipboard.writeText(props.source)
    copied.value = true
    window.setTimeout(() => { copied.value = false }, 2000)
  } catch (err) {
    console.warn('[CopyButton] clipboard write failed:', err)
  }
}
</script>

<template>
  <button
    type="button"
    class="copy-button"
    :class="{ 'is-copied': copied }"
    :aria-label="'Copy code'"
    @click="copy"
  >
    {{ copied ? 'Copied' : 'Copy' }}
  </button>
</template>
```

No `<style>` block in the SFC — all visuals via `style.css` to honor the .slidev-layout scoping rule.

  <action>
Create `setup/main.ts`. This is a Slidev integration point: it runs once per app mount and receives `{ app, router }`. We use the router's `afterEach` hook to attach copy buttons after every slide navigation (including initial route).

The injection strategy:
1. After each route change, wait one tick (`requestAnimationFrame` x2, or `setTimeout(..., 50)` — pick `requestAnimationFrame` nested twice for reliability) so Shiki's `<pre>` blocks have rendered.
2. Query `document.querySelectorAll('.slidev-layout pre[class*="shiki"]')`.
3. For each `pre` that does NOT already have `data-copy-mounted="true"`:
   - Set the data attribute to mark it.
   - Create a `<span class="copy-button-host">` and append it to the pre.
   - Mount a `CopyButton` Vue app into that span, passing the pre's `<code>.textContent ?? pre.textContent ?? ''` as the `source` prop.
   - Save the unmount function on the host element so we don't leak (`(host as any).__unmount = appInstance.unmount`).

Use `createApp` from `vue` (not the host app) for each button so each has isolated state. This is intentional — the host app's plugins aren't needed for a stateless button.

Exact file content:

```ts
import { createApp, h } from 'vue'
import { defineAppSetup } from '@slidev/types'
import CopyButton from '../components/CopyButton.vue'

const MARK = 'data-copy-mounted'

function mountCopyButtons() {
  const blocks = document.querySelectorAll<HTMLPreElement>(
    '.slidev-layout pre[class*="shiki"]'
  )
  blocks.forEach((pre) => {
    if (pre.getAttribute(MARK) === 'true') return
    pre.setAttribute(MARK, 'true')

    const host = document.createElement('span')
    host.className = 'copy-button-host'
    pre.appendChild(host)

    const code = pre.querySelector('code')
    const source = (code?.textContent ?? pre.textContent ?? '').replace(/\n$/, '')

    const app = createApp({
      render: () => h(CopyButton, { source }),
    })
    app.mount(host)
  })
}

export default defineAppSetup(({ router }) => {
  router.afterEach(() => {
    requestAnimationFrame(() => {
      requestAnimationFrame(() => {
        mountCopyButtons()
      })
    })
  })

  // Initial pass once the document is ready (router.afterEach does NOT
  // fire for the first nav in some Slidev versions).
  if (typeof window !== 'undefined') {
    if (document.readyState === 'complete') {
      requestAnimationFrame(() => requestAnimationFrame(mountCopyButtons))
    } else {
      window.addEventListener('load', () => {
        requestAnimationFrame(() => requestAnimationFrame(mountCopyButtons))
      })
    }
  }
})
```

Why `.slidev-layout pre[class*="shiki"]` (not bare `pre`):
- Excludes `<pre>` inside presenter chrome (`.slidev-presenter`) — D-14 contract.
- Excludes overview/grid thumbnails — same reason.

Why strip trailing newline: Shiki's `<pre>` includes a trailing `\n` from the markdown fence; stripping it gives clean clipboard output.

Why nested `requestAnimationFrame`: Shiki block hydration on slide change can lag one frame behind route resolution.
