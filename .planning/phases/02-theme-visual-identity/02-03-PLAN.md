```css
/*
 * syai-claude-workshop — theme overlay
 *
 * Scoping contract (D-14, D-15): every selector below EXCEPT `:root`
 * MUST have `.slidev-layout` as an ancestor. Presenter chrome
 * (.slidev-presenter, sidebar thumbnails) MUST NOT inherit these styles.
 *
 * Palette (D-01) — mirrored from uno.config.ts shortcuts so that CSS
 * rules below can use var(--token) without depending on UnoCSS bracket
 * notation inside selector bodies.
 */

:root {
  --bg-main:      #1A1714;
  --surface-code: #14110E;
  --text-main:    #F4ECE0;
  --text-dim:     rgba(244, 236, 224, 0.65);
  --accent:       #CC785C;
}

/* Slide canvas — single-tone background per D-02. */
.slidev-layout {
  background-color: var(--bg-main);
  color: var(--text-main);
}

/* Headings render in cream per D-04 — NOT rust. */
.slidev-layout h1,
.slidev-layout h2,
.slidev-layout h3,
.slidev-layout h4,
.slidev-layout h5,
.slidev-layout h6 {
  color: var(--text-main);
}

/*
 * Code-block frame (D-05):
 *   - Tinted panel via --surface-code
 *   - 2px solid rust left rule, full block height
 *   - SHARP corners — no border-radius (architectural, not card-like)
 *   - No outer border, no shadow
 *   - Internal padding 0.9rem 1.1rem
 *
 * Selector targets Shiki's <pre class="shiki ..."> wrapper. The
 * `[class*="shiki"]` matches any Shiki theme variant (e.g., shiki vitesse-dark).
 */
.slidev-layout pre[class*="shiki"] {
  background-color: var(--surface-code) !important;
  border-left: 2px solid var(--accent);
  border-radius: 0;
  border-top: 0;
  border-right: 0;
  border-bottom: 0;
  box-shadow: none;
  padding: 0.9rem 1.1rem;
}

/*
 * Inline code (D-07):
 *   - JetBrains Mono inherited from Slidev fonts.mono
 *   - Subtle pill on --surface-code background
 *   - NO left rule (inline ≠ block)
 *
 * `:not(pre > code)` ensures we only style inline code, not the <code>
 * inside Shiki's <pre>. The :not(.shiki) guard is belt-and-braces.
 */
.slidev-layout :not(pre) > code:not(.shiki) {
  background-color: var(--surface-code);
  color: var(--text-main);
  padding: 0.1em 0.35em;
  border-radius: 0;
}

/* Dim text utility — opt-in via class, mirrors UnoCSS text-dim. */
.slidev-layout .text-dim {
  color: var(--text-dim);
}
```

Notes on the locked rules:
- The five `:root` variables are copied verbatim from D-01.
- `.slidev-layout pre[class*="shiki"]` is the canonical selector for Shiki-rendered fenced blocks in Slidev. It survives Shiki theme name changes (vitesse-dark, github-dark-default, etc.).
- `!important` on `background-color` is needed because Shiki injects the theme background as an inline style on `<pre>`. Without `!important`, our tinted `--surface-code` is overridden by the inline. This is a documented pattern for Shiki theme overlays.
- The four explicit zero borders (top/right/bottom/0) plus the explicit `border-left` produce a clean single-side rule without inheriting any default `border` shorthand from the theme.
- `border-radius: 0` is enforced both on the code frame and the inline pill per D-05 ("sharp corners — architectural").
- Inline-code rule uses `:not(pre) > code` so it never matches Shiki's inner `<code>` (which lives inside `<pre>`).
- The `.text-dim` rule is intentionally redundant with the UnoCSS shortcut; having it here lets non-UnoCSS markup (e.g., raw HTML in slides) reach the same token.

Do NOT:
- Add any selector without `.slidev-layout` as ancestor (except the `:root` block). Specifically forbidden: bare `body`, bare `pre`, bare `code`, bare `h1`, bare `*`. ANY of those leaks into `/presenter/` chrome and FAILS D-15.
- Add `@import url(...)` for Inter or JetBrains Mono — Slidev `fonts:` in 02-01 handles fetching. A second import wastes bandwidth and bypasses Slidev's font-display config.
- Add `themeConfig` references — that's a Slidev frontmatter concern, not CSS.
- Style `.slidev-presenter`, `.slidev-overview`, or anything outside `.slidev-layout` — those are PRESENTER chrome (D-15).
- Add slide transitions, animations, or custom layout selectors (`.layout-section`, etc.) — out of scope for Phase 2.
- Add line-number CSS (`.line-numbers`, `pre[lineNumbers]`) — D-06 keeps line numbers off by default; per-slide opt-in stays functional via Slidev's built-in handling.
- Use any hex literal NOT in D-01. The five values `#1A1714`, `#14110E`, `#F4ECE0`, `rgba(244,236,224,0.65)`, `#CC785C` are the only colors permitted.
