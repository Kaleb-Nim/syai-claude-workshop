The file MUST be exactly:

```ts
import { defineConfig } from 'unocss'

/**
 * UnoCSS shortcut tokens for syai-claude-workshop.
 *
 * Palette anchors (D-01, locked in 02-CONTEXT.md):
 *   --bg-main:      #1A1714  warm near-black slide canvas
 *   --surface-code: #14110E  ~3% darker, code-block panel only (D-02, D-05)
 *   --text-main:    #F4ECE0  warm cream body / heading text
 *   --text-dim:     rgba(244, 236, 224, 0.65)  secondary cream
 *   --accent:       #CC785C  rust — code-block left rule + opt-in (D-03, D-04)
 *
 * Per D-08, every color used in slides.md or style.css MUST flow through
 * one of these shortcuts. No inline hex literals in slides.
 */
export default defineConfig({
  shortcuts: {
    // Backgrounds
    'bg-main': 'bg-[#1A1714]',
    'bg-surface-code': 'bg-[#14110E]',

    // Text
    'text-main': 'text-[#F4ECE0]',
    'text-dim': 'text-[rgba(244,236,224,0.65)]',
    'text-accent': 'text-[#CC785C]',

    // Borders (consumed by code-block left rule in 02-03)
    'border-accent': 'border-[#CC785C]',
  },
})
```

Notes on the locked values:
- The five hex/rgba values are copied verbatim from D-01 — do NOT round, recolor, or "simplify."
- UnoCSS bracket notation (`bg-[#1A1714]`) is the correct way to encode arbitrary hex values inside a shortcut. Reference: https://unocss.dev/config/shortcuts and https://unocss.dev/config/rules#bracket-notation .
- `text-dim` uses `rgba(...)` per D-01 (not a separate cream-65% hex); UnoCSS bracket notation accepts the rgba string with no spaces (commas are fine).
- `border-accent` only sets the `border-color` token; the `2px solid` left-rule width and side are applied by `style.css` in 02-03 — keep responsibilities split.
- Do NOT add `themeConfig.primary`, presets, transformers, or content scanning blocks. Slidev's bundled UnoCSS already configures presets — adding our own can conflict. Shortcuts only.
- Do NOT install any UnoCSS package via `bun add` — it ships with Slidev. The `unocss` import resolves through Slidev's bundled dep.

Do NOT:
- Use TypeScript types beyond what `defineConfig` infers (no manual `Shortcut[]` typing — let UnoCSS infer).
- Add named exports (Slidev expects a default export).
- Place the file under `setup/`, `styles/`, or `.slidev/` — root only, per Slidev convention for `uno.config.ts`.
- Add `theme.colors` (use shortcuts, not theme overrides — per D-08 the contract is shortcut names).

After writing, confirm `bunx slidev build` exits 0. UnoCSS will warn (not error) on unused shortcuts — that is expected; they get consumed in 02-03 and (optionally) future slides.
