Create `vercel.json` at the repo root with EXACTLY this content (per CLAUDE.md authoritative spec + DEPL-02, DEPL-03):

```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "framework": null,
  "installCommand": "bun install",
  "buildCommand": "bunx slidev build",
  "outputDirectory": "dist",
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

Hard requirements (do not deviate):
- `framework: null` — Slidev outputs a Vite SPA, NOT a Vue/Nuxt project; using the Vue preset breaks build (per CLAUDE.md "What NOT to Use").
- `installCommand: "bun install"` — explicit even though Vercel auto-detects `bun.lockb`; pinning per CLAUDE.md.
- `buildCommand: "bunx slidev build"` — Bun-runs the Slidev CLI; produces `dist/`.
- `outputDirectory: "dist"` — matches Slidev's default build output.
- `rewrites: [{ "source": "/(.*)", "destination": "/index.html" }]` — SPA fallback so deep-link routes (e.g. `/3`) hit `index.html` and Slidev's hash/history router resolves them client-side. Without this, hard-refresh on `/3` returns 404 (DEPL-04 fails).

Do NOT add: `headers`, `redirects`, `functions`, `crons`, `regions`, or any `env` block. None are needed for this static deck.

Do NOT use `routes:` (legacy Vercel config — `rewrites:` is the modern equivalent).
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local CLI → Vercel API | `bunx vercel` calls cross trust boundary via the user's existing auth token (managed by Vercel CLI; never passed in repo). |
| GitHub repo → Vercel | After `vercel git connect` (or manual link), Vercel pulls source from GitHub. Standard Vercel↔GitHub OAuth scope. |
| Public internet → `*.vercel.app` | Anyone with the URL can view the deck. Intended — public workshop scaffold. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Information Disclosure | `.vercel/project.json` | mitigate | Gitignored in Plan 01 Task 3; never committed. Contains org/project IDs only (no secrets), but still excluded as a discipline. |
| T-01-05 | Spoofing | Production URL | accept | `*.vercel.app` subdomains are managed by Vercel; squatting handled by the platform. No custom domain this phase. |
| T-01-06 | Tampering | `vercel.json` | mitigate | Committed and reviewed; SPA rewrite is the only routing rule, no headers/redirects/functions to abuse. |
| T-01-07 | Information Disclosure | Deck contents | accept | Public by design — no secrets, no PII, no auth state in the static SPA. Phase 1 deck is 3 placeholder slides; instructor authors content post-handoff. |
| T-01-08 | DoS | Vercel free tier | accept | Workshop traffic is well within free-tier limits per CLAUDE.md. No mitigation needed for v1. |
</threat_model>
